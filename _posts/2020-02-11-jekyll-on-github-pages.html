Jekyll

Jekyll - генератор статических сайтов. Вы даете ему текст, написанный на вашем любимом языке разметки, и он использует макеты для создания статического сайта. Вы можете настроить внешний вид URL-адресов сайта, данные, отображаемые на сайте, и многое другое.

Jekyll имеет очень подробную и простую документацию, с которой можно ознакомится ТУТ.

Jekyll позволяет вам используя понятную структуру создавать различного рода простые сайты. Почему только простые? Причина проста - вы можете упереться в технические ограничения статического генератора и GitHub Pages и это не позволит вам реализовать желаемые функционал. Не стоит забывать, что мы рассматриваем вариант, что у вас нет сервера и ваш сайт хостится на GitHub.

При необходимости сайт на Jekyll можно будет достаточно легко перенести на что-то более производительное и с большим функционалом, возможно какую-либо CMS, но тогда ваш сайт потеряет в производительности по сравнению с Jekyll.

Быстрый старт на Jekyll

Существует вариант из документации - вы получите базовую структуру, которую сможете доработать под свои нужды и вариант с заимствованием уже готовой jekyll темы с последующим развертыванием ее. Мы рассмотрим оба варианта.

Вариант из документации с получением "базового" сайта

  1. Установите срезу разработки Ruby -> ТЫК
  2. Установите Jekyll и гемы gem install jekyll bundler
  3. Создайте новый сайт Jekyll jekyll new myblog
  4. Перейдите в новый каталог myblog cd myblog
  5. Создайте сайт и запустите локальный сервер bundle exec jekyll serve
  6. Перейдите по адресу http://localhost:4000
  7. Далее его можно залить на GitHub:
    • Инициализируем репозиторий git init
    • Добавляем файлы в индекс git add .
    • Коммитим git commit -m "initial commit"
    • Указываем origin для вашего репозитория git remote add origin https://github.com/navasiolau/username.github.io.git
    • Пушим наши файлы в мастерgit push -u origin master
    • Ждем некоторое время, пока github билдит наш сайт, после чего он будет доступен по адресу username.github.io.git

Вариант с использованием готовой темы

  1. Переходим на маркет с темами в бесплатный раздел -> ТЫК
  2. Выбираем тему
  3. Переходим в Github репозиторий темы
  4. Скачиваем ее
  5. Удаляем .git папку в корне, изменяем содержание под свои нужды
  6. Заливаем изменения на Github

Заключение

После выполнения одного из вариантов ваш сайт должен быть доступен по адресу username.github.io. Это прекрасно, но хотелось бы, чтобы он был доступен по "человеческому" адресу. Об этом и будет следующая статья.
