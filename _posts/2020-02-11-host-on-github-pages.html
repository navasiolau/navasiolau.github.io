Github Pages

Github Pages - это хостинг от Github, позволяющий публиковать статические веб-сайты. Вы не сможете использовать backend - это главное ограничение хостинга от гитхаба, но при этом вы можете использовать генераторы статики и их немало, а еще для них есть плагины частично реализующие необходимую функциональность.

Доступные для использования генераторы статики

  • jekyll
  • Getsby
  • Hugo
  • Hexo
  • Slate

jekyll - самый распространённый, наиболее удобный из-за своей простоты, наличия большого количества шаблонов и плагинов. Стоит отметить что не все плагины поддерживаются нативно, лишь основные, поэтому использование не поддерживаемых потребует от вас немного попотеть (если вам это конечно будет нужно).

Фактически, если вы хотите завести личный блог или сделать страницу о себе, партфолио или что-то подобное, то github pages это хороший вариант.

Преимущества хостинга Github

  • Бесплатный хостинг
  • Короткое время отклика сервера
  • Бесплатный ssl сертификат по умолчанию
  • Бесплатный домен вида username.github.io
  • Удобная панель управления
  • Возможность нативно использовать все преищусества github

Ограничения GitHub Pages

  • Сайт не может быть более 1 Гб размером
  • Рекомендованный замер GitHub репозитория - 1 Гб
  • На сайтах GitHub Pages установлен мягкий предел пропускной способности 100 ГБ в месяц.
  • Сайты GitHub Pages имеют мягкий предел 10 билдов в час.

В случае нарушения этих ограничений администрация GitHub пришлёт вам вежливое письмо с просьбой придерживаться рекомендаций.

Последовательность действий для публикации сайта на GitHub Pages

  • Создать репозиторий в своем аккаунте гитхаб с названием username.github.io (В качестве username используйте свой никнейм в Github).
  • Создать файл index.html и добавить его в git репозиторий, после чего закоммитить изменения и запушить в мастер. Либо внести файлы сайта, если у вас он уже свёрстан заранее, либо если вы используете шаблон.
  • Перейти по адресу https://username.github.io - Вы великолепны!
